Overview of files

Download the provided files here: tuning-in-C.zip

Most of the provided files are here for convenience and there is nothing for you to do. The only file in which you are being asked to write any code is tune.c.

Provided files

  • tune.c is where you will write your code. Detailed instructions are found inside that file.
  • tune.h is the header file for tune.c. This is definitely not intended to be an important part of this "homework" but I will provide some info: Header files allow us to write code that uses functions that are implemented in a different .c file. Technically we don't even need access to the file tune.c in order to get gcc to "assemble" a program that includes tune.h. But to get gcc to actually produce an executable, we will need to provide the implementation of tune.h (either as source code or as a .o file). One nice thing this means is that you can do things like swap out implementations of a header file at "link time" without having to recompile your code. It is a lovely little layer of indirection that allows for all kinds of nice code reuse.
  • verify-solution.c is the program that you will run to test your code. It is compiled and linked using the Makefile. The executable is saved to bin/tune.
  • Makefile is a "build" tool that allows us to automate certain tasks relating to compiling. You don't need to read or understand this file, but its existence offers you some conveniences: normally you would need to manually invoke gcc to produce an executable. Thanks to Makefile, if you open a terminal in the same directory as the Makefile, you can instead use these commands:
    • make: this will compile and link the code, saving the executable to the file bin/tune i.e. the executable is named tune and it is saved inside the bin directory.
    • make test: same as make plus it runs bin/tune
    • make clean: this deletes all binaries and build artifacts to give you a fresh start

Some details about the twelve_TET_octave function

You should read the other files for the full details, but I want to give an example here of exactly how the function should behave.

You are writing a function for someone else to use. The example below shows how someone else's code might use your function.

If someone were to define an array called notes and then call the function as follows:

#include "tune.h"

#define OCTAVE_WIDTH 12

double notes[OCTAVE_WIDTH + 1];
twelve_TET_octave(220.0, notes);

Then the values in their notes array should be the following (note that I have truncated the last few decimal places for readability):

[ 220.00,  233.08,  246.94,  261.63,  277.18,  293.66,  311.13,
  329.63,  349.23,  369.99,  392.00,  415.30,  440.00 ]

It is your job to populate this array. Notice that there are thirteen elements in the array even though an octave is only twelve notes. This is because the root appears twice, once at the start and once at the end. This should allow you to quickly check the correctness of your output: the final element should always be twice the first element.

Mathematical details of 12-tone equal temperament

You will need to use the library math.h which is already included in tune.c because computing exponents is required in order to perform 12-tone equal temperament. Exponents are computed using the function pow(base, exponent). For example, 102 would be computed as pow(10, 2).

In twelve-tone equal temperament, the ratio between the frequencies of two adjacent semitones is the twelfth root of two, which is the same as two raised to the power of 1/12. In C code, that is pow(2, 1/12.0).

This means that given any frequency f, in order to find the frequency that is one semitone higher than f, we simply multiply f by pow(2, 1/12.0).
